Detailed Description

template<uint8_t N, class R, class T>
class ghttp::RouteManager< N, R, T >

Routemanager class used to guide processing of requests and handling failures along with servicing request/response framework.

Note
The last register route should be your unhandled route handler if you want to deal with this situation
Template Parameters
NRoute
RClass of route managed by this instance
TClass of your application specific shared data
Examples
ghttp_ethercard_get_with_routes.ino, ghttp_ethercard_server_authorization.ino, ghttp_ethercard_session_login.ino, ghttp_ethercard_windoze_toast_notification.ino, ghttp_static_pre_route_request_response.ino, ghttp_static_redirect_route_request_response.ino, ghttp_static_request_response_session_simple.ino, ghttp_static_request_response_simple.ino, and ghttp_static_unhandled_route_request_response.ino.

Register route.

The new route can be a normal route, the last normal route could accept any route to handle the condition where no other route accepted request and finally it could be a pre-route handler. A pre-route handler makes sure semantics of the request are generally what your application accepts and can setup other globals to aid your other requests.

Parameters
[in]rRoute pointer Make sure your routes are global, don't create them on stack
[in]pre_route_checkFlag indicating this route is the pre-route check, default false => normal route
Returns
State
Return values
+nRoute count after addition
-1Failed. Make sure N is larger enough for your application
